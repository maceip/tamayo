four ways to prove enough, and no more

The crypto packages stay low level, but the token service now exposes four rows product runtimes can wrap: one-use access, private continuity, policy-bound email, and Google email verification token compatibility.

  1. Burn token Anonymous one-use pass. The verifier learns that this request was allowed once, then burns the token.
  2. Private identity token Same visitor for one relying party, no email shown. The repeated name only works for that site.
  3. Policy-bound email token Email address is shown, and issuance can require a trusted runtime measurement.
  4. Google email verification token Email address only, using the Google-compatible format for interop and regression tests.
